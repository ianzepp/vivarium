# Phase 06: Incremental Operations

## Interpreted Problem

Phase 05 exposed a usable local agent interface. Phase 06 makes the archive
maintainable as new mail arrives: a scheduled run should pull new Proton Bridge
mail and refresh derived local metadata without turning Vivi into a long-running
mail client or daemon.

The current repo already has an incremental ingest command, `vivi sync`, because
IMAP sync downloads only missing messages. This phase extends that command so it
also refreshes local derived artifacts after ingest.

## Phase Spec

### Goal

Make the archive maintainable as new mail arrives without turning Vivarium back
into a general mail client.

### Inputs

- read-only Proton Bridge ingest
- catalog
- extraction
- search/embedding indexes

### Expected Outputs

1. incremental ingest command
2. incremental catalog update
3. incremental extraction update
4. incremental search/embedding update
5. operational docs for scheduled local runs
6. corruption recovery and rebuild documentation

### Out Of Scope

- SMTP/send
- cloud sync service
- always-on multi-user daemon
- mobile support
- semantic embeddings until a local embedding index exists

### Checkpoint Target

A scheduled local run can pull new Proton Bridge mail and update catalog,
extraction, keyword search, and embeddings without reprocessing the full archive.

## Workstreams

### WS-06-A: Sync Updates Derived State

- Keep `vivi sync` as the incremental ingest command.
- After remote sync, scan Maildir paths and catalog only uncataloged local
  messages.
- Report new remote downloads and derived update counts.

### WS-06-B: Incremental Extraction

- Run extraction for newly cataloged messages.
- Count extraction successes and errors.
- Do not re-extract the whole archive on every scheduled run.

### WS-06-C: Search/Embedding State

- Document keyword search as live over local Maildir for now, so it becomes
  current as soon as raw messages are present.
- Keep semantic embeddings explicitly unsupported until a local embedding index
  exists.

### WS-06-D: Operational Docs

- Document a launchd/cron-friendly scheduled command.
- Document cache repair:
  - raw `.eml` remains source of truth
  - `.vivarium/catalog.json` can be deleted and regenerated by running sync
  - parse errors should be investigated against the raw message path

### WS-06-E: Tests

- Add tests for incremental catalog update skipping already-cataloged paths.
- Add tests for extraction over newly cataloged entries.
- Run full repo tests and a live local sync probe.

## Verification Commands

```sh
cargo test
cargo run --quiet -- sync --account personal-proton --limit 0
```

Use a low or zero sync limit for live validation to avoid downloading a large
batch while still exercising local derived-state refresh.

## Gate Plan

| Gate | Trigger | Pass Criteria | Fail Action |
|------|---------|---------------|-------------|
| Build + Tests | After implementation | `cargo test` passes | Fix compilation or test failures |
| Local Ops Probe | After build/tests | `vivi sync --limit 0` reports catalog/extraction counts without downloading a batch | Fix incremental update path |
| Checkpoint | Before commit | Scheduled sync updates raw mail and derived local state without reprocessing the full archive | Revise sync/update split |

Phase 06 completed.
